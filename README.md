# A/B Test Analysis: Fast Food Marketing and Cookie Cats

**Module 2 . Sprint 2: A/B Testing** | Turing College . Data Science Programme

Two end-to-end A/B test analyses with statistical rigour and business-oriented recommendations.

---

## Project Goal

Evaluate experimental results from two real-world datasets, estimate treatment effects with confidence intervals and hypothesis tests, and deliver actionable decisions for stakeholders.

---

## Experiments

### 1. Fast Food Marketing Campaign

A fast food chain tested three promotions across stores for four weeks.

| Property | Value |
|---|---|
| File | `data/raw/WA_Marketing-Campaign.csv` |
| Design | 3-arm experiment, one promotion per store, 4 weekly observations |
| Primary metric | Store-level average weekly sales (thousands $) |
| Tests | Welch t-test (pairwise), global A/B/C test, Holm correction |

### 2. Cookie Cats Mobile Game

Cookie Cats moved its first forced-wait gate from level 30 to level 40.

| Property | Value |
|---|---|
| File | `data/raw/cookie_cats.csv` |
| Design | 2-arm experiment, random assignment at install |
| Primary metric | Day 7 retention |
| Secondary metric | Engagement (sum_gamerounds) |
| Tests | Two-proportion z-test, Welch t-test, Mann-Whitney U, bootstrap CI |

## Notebook Flow

### Notebook 1: Fast Food Marketing

| Section | Key content |
|---|---|
| 1. Imports and Setup | Dependencies, constants |
| 2. Problem Definition | Target metric with STEDII rationale, statistical standards |
| 3. Data Loading | Shape, columns |
| 4. Data Cleaning | Missing values, duplicates, type coercion |
| 5. Metric Construction | Store-level aggregation (avoids repeated-measures violation) |
| 6. SRM Check | Chi-square test for sample ratio mismatch |
| 7. Covariate Balance | Store age and market size by promotion |
| 8. EDA | Bar charts, violin plots, weekly trend (all with numeric labels) |
| 9. Pairwise A/B Tests | Promo 1 vs 2, 1 vs 3 with Welch + bootstrap |
| 10. A/B/C Global Test | Omnibus test + Holm-corrected post-hoc |
| 11. Business Impact | Dollar-denominated revenue projection |
| 12. Decision | Stakeholder recommendation |
| 13. Limitations | Short window, no cost data, randomization assumptions |

### Notebook 2: Cookie Cats

| Section | Key content |
|---|---|
| 1. Imports and Setup | Dependencies, constants |
| 2. Problem Definition | Metric framework (primary, guardrail, secondary), statistical standards |
| 3. Data Loading | Shape, columns |
| 4. Data Cleaning | Missing values, duplicate users, type coercion |
| 5. SRM Check | Chi-square test for 50/50 split |
| 6. Exposed Group | Filter to players who interacted with the feature |
| 7. EDA | Retention bars, engagement violins, absolute/relative quantification |
| 8. Retention Tests | Two-proportion z-test on exposed + full sample |
| 9. Engagement Tests | Welch t-test, bootstrap CI, Mann-Whitney U |
| 10. Business Impact | Revenue projection from retention lift |
| 11. Decision | Keep gate at level 30 |
| 12. Limitations | Short retention window, no revenue data, exposed group proxy |

## How to Run

```bash
# 1. Install
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install -e .

# 2. Launch notebooks
make notebook
# or: jupyter lab notebooks/
```

Both notebooks run top-to-bottom. All random seeds are fixed via `RANDOM_SEED = 42`.

---

## Reproducibility

| Mechanism | Detail |
|---|---|
| Fixed seed | `RANDOM_SEED = 42` for all stochastic operations |
| Bootstrap | 20,000 resamples with fixed seed |
| Cleaning | Explicit, reproducible pipeline with CleaningReport |
| Multiple comparisons | Holm correction for pairwise tests |
